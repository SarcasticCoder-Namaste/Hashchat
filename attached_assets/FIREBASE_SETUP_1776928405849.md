# Firebase Hosting Setup Guide

## Quick Start

### 1. Create Firebase Project
- Go to [Firebase Console](https://console.firebase.google.com)
- Click "Add project"
- Enter your project name
- Complete the setup

### 2. Get Your Project ID
- In Firebase Console, go to Project Settings
- Copy your Project ID
- Replace `YOUR_FIREBASE_PROJECT_ID` in `.firebaserc`

### 3. Install Firebase CLI
```bash
npm install -g firebase-tools
```

### 4. Login to Firebase
```bash
firebase login
```

### 5. Initialize Firebase (if not done)
```bash
firebase init
```

### 6. Build Your App
```bash
npm run build
```

### 7. Deploy to Firebase
```bash
firebase deploy
```

## Backend Deployment

For the Express backend, you have two options:

### Option A: Use Replit's Built-in Publishing (Recommended)
- Simpler setup
- No additional configuration needed
- Just click "Publish" button in Replit

### Option B: Deploy Backend Separately
- Use Google Cloud Run or Firebase Cloud Functions
- More complex setup
- Requires backend URL configuration in frontend

## Frontend-Only Deployment
If deploying only the frontend to Firebase Hosting:

1. Build: `npm run build`
2. Deploy: `firebase deploy --only hosting`

**Note:** Your backend API calls will need to point to a backend server. Either:
- Use Replit's published URL as your API endpoint
- Deploy backend to Cloud Run/Functions
- Use the `.env` file to configure API endpoints

## Environment Variables
Create `.env.production` for production builds:
```
VITE_API_URL=https://your-backend-url.com
```

## Troubleshooting

### Build fails
- Run `npm install` to ensure all dependencies
- Check Node.js version: `node --version` (should be 16+)

### Deploy fails
- Ensure `.firebaserc` has correct Project ID
- Run `firebase login` again
- Check you have Firebase CLI permissions

### API calls don't work
- Verify backend URL in environment variables
- Check CORS settings on backend
- Ensure backend is running
